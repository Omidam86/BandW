# BandW

بلاگ «BandW» — فضای مینیمال و سیاه‌وسفید برای خواندن و نوشتنِ متن‌های عمیق.

این پروژه یک وب‌سایت استاتیک/SPA ساده به زبان فارسی است که تمرکز اصلیش روی متن و خواناییه. هدف: کم کردن حواس‌پرتی، تأکید بر محتوا، و تجربه کاربری روان روی موبایل و دسکتاپ.

---

## چه‌چیزی اینجا دیده می‌شه؟
- طراحی مینیمال و تاریک (Black & White) با تمرکز کامل روی متن
- صفحه‌ی اصلی با فید مقالات، نمایش برگزیده‌ها و پر بازدیدها
- ارسال پست توسط کاربران، و پنل مدیریت برای تایید، ویرایش و دسته‌بندی
- جستجوی ساده داخل پست‌ها و دسته‌بندی‌ها
- پیش‌نمایش مقاله و نمایش مقالات مشابه

---

## ایده‌ی اصلی
BandW برای کسانی ساخته شده که می‌خوان بدون حواس‌پرتی بنویسند و بخوانند — بدون تصویر، بدون تبلیغ، فقط کلمه. تجربه کاربری ساده ولی دقیق طراحی شده تا حتی روی شبکه‌های کند و موبایل هم سریع و روان باشه.

---

## فیچرهای قابل توجه
- کارت‌های مقاله با خلاصه و متادیتا (تعداد بازدید، دسته)
- صفحه ارسال مقاله (کاربر ارسال می‌کنه، مقاله بعداً توسط ادمین منتشر می‌شه)
- پنل مدیریت سبک و سریع برای ادمین (بازبینی، انتشار، ویرایش، حذف، مدیریت دسته‌ها)
- رابط کاربری راست‌چین و مناسب زبان فارسی

---

## راه‌اندازی محلی (برای توسعه‌دهنده)
این پروژه با فایل‌های HTML/CSS/JS ساده ساخته شده و برای راه‌اندازی محلی نیاز به یک سرور استاتیک یا Live Server دارید.

1. مخزن را کلون کنید:

```bash
git clone https://github.com/Omidam86/BandW.git
cd BandW
```

2. یک سرور استاتیک ساده اجرا کنید (مثال با `http-server` یا `live-server`):

```bash
# اگر npm دارید:
npx http-server -c-1 .
# یا
npx live-server .
```

3. صفحات را در مرورگر باز کنید:
- `index.html` برای سایت
- `submit.html` برای ارسال مطلب
- `admin.html` برای پنل مدیریت


> توجه: پروژه برای ذخیره‌سازی و تعامل با دیتابیس از Supabase استفاده می‌کند — برای عملکرد کامل باید مقادیر SUPABASE_URL و SUPABASE_KEY را تنظیم کنید یا یک نمونه تستی Supabase داشته باشید.

---

## برای توسعه‌دهنده‌ها — نکات سریع
- فایل‌های اصلی: `index.html`, `submit.html`, `admin.html` و CSS درون‌خطی داخل هر فایل
- منطق ارتباط با Supabase در اسکریپت‌های داخلی قرار داره — برای توسعه بهتر می‌تونید این کدها رو به ماژول‌های جدا منتقل کنید
- برای اضافه کردن فیچرها (مثلاً pagination، جستجوی پیشرفته، فیلترها) بهترین نقطه شروع: تابع `getPosts` و بخش render در `index.html` و منطق ارسال/بارگذاری در `submit.html` و `admin.html`.

## نحوه مشارکت
اگر علاقه‌مندید کمک کنید:
1. یک issue باز کنید یا یک feature request پیشنهاد بدید
2. روی یک شاخه جدید کار کنید: `feature/short-description`
3. Pull request ارسال کنید و در توضیحات PR موارد زیر را ذکر کنید:
   - کاری که انجام شده
   - صفحات/فایل‌های تغییر یافته
   - دستورالعمل تست

## Roadmap کوتاه‌مدت
- بهبود سرچ (شامل متن پست و دسته‌بندی، جستجوی حساس به حروف کوچک/بزرگ)
- Pagination / بارگذاری تنبل برای بهبود عملکرد فید
- نمایش اطلاعات تکمیلی هر پست: زمان خواندن، خلاصه، نویسنده و تگ‌ها
- بهینه‌سازی UI برای موبایل و اضافه کردن skeleton loaders
